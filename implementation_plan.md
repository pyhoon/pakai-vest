# Implementation Plan: Pakai-Vest CRUD Extension for B4J Pakai Server

The goal is to design and build `Pakai-Vest`, a new B4J extension/library packaged as `Vest.b4xlib`. It will provide four customizable custom class templates to quickly add database tables, HTML CRUD lists, and REST API endpoints to any Pakai Server project.

## Open Questions

> [!NOTE]
> Before we start executing the implementation, please review the proposed plan and clarify:
> 1. **Placeholder Entity Name**: We plan to use `items` / `tbl_items` and `item` placeholders in the CRUD templates (comparable to `products`/`categories` in Pakai-Lanyard). Would you prefer a different entity name (e.g. `items`, `records`, `entities`)?
> 2. **Included Fields**: We propose standard template fields for the generic CRUD form: `id`, `code`, `name`, and `description`/`price`. Let us know if you want other/fewer default template fields.

## Proposed Components and Class Templates

We will model `Pakai-Vest` after `Pakai-Lanyard`, using a consistent project structure and packaging workflow. The extension will offer the following templates:

1. **CrudModel (`CrudModel.bas`)**:
   - Manages table lifecycle and operations using `MiniORM`.
   - Includes template routines for `Create`, `Read` (search/filter), `Update`, `Delete`, `GetRowById`, and database table initialization.

2. **CrudView (`CrudView.bas`)**:
   - Programmatically generates responsive CRUD tables and modal forms (Add, Edit, Delete) using `MiniHtml` DSL and Bootstrap 5.
   - Renders search filters and triggers HTMX actions.

3. **CrudWebHandler (`CrudWebHandler.bas`)**:
   - Maps HTML web endpoints (e.g. `/hx/items/*`) using `EndsMeet`.
   - Interacts with the view and model to serve partial updates via HTMX and toasts.

4. **CrudApiHandler (`CrudApiHandler.bas`)**:
   - Implements JSON/XML REST API endpoints (e.g. `/api/items`) utilizing `WebApiUtils` for standard payload validation and responses.

## Technical Details

### 1. `manifest.txt`
Declares the author, version, and the four templates:
```ini
Author=Aeric
Version=0.10
CustomClassTemplates=Crud Model|CrudModel,Crud View|CrudView,Crud Web Handler|CrudWebHandler,Crud Api Handler|CrudApiHandler
Supported Platforms=B4J
```

### 2. `convert.bat`
Automates conversion of classes into excluded files before zipping:
```bat
copy ..\CrudModel.bas ..\CrudModel.b4x_excluded
copy ..\CrudView.bas ..\CrudView.b4x_excluded
copy ..\CrudWebHandler.bas ..\CrudWebHandler.b4x_excluded
copy ..\CrudApiHandler.bas ..\CrudApiHandler.b4x_excluded
```

### 3. Packaging Macros in `Pakai-Vest.b4j`
Matches the exact build automation setup of `Pakai-Lanyard`:
- **Convert classes**: Runs `convert.bat`
- **Update Manifest**: Executes manifest writer to keep versions synced
- **Create B4xLib**: Jars the files into `release/Vest.b4xlib`
- **Copy to AddLibs**: Copies the generated `.b4xlib` to B4J's shared libraries path

## Implementation Details for Templates

### CrudModel (`CrudModel.bas`)
A generic `MiniORM` database client that developers can customize by replacing the table (`tbl_items`) and its columns (`item_code`, `item_name`, etc.):
- `CreateUsersTable` (adapted to `CreateItemsTable`)
- `Create`, `Read`, `Update`, `Delete` methods
- `GetRowById`, `FindRowByCode` helpers

### CrudView (`CrudView.bas`)
A programmatic HTML generator using `MiniHtml` to render:
- Responsive Bootstrap 5 search layout
- Data table row with Edit/Delete icons pointing to HTMX endpoints
- Add, Edit, Delete modals with inputs mapping fields
- OOB (Out-of-band) Toast messages and Custom Event dispatch scripts

### CrudWebHandler (`CrudWebHandler.bas`)
Serves CRUD requests by calling `EndsMeet`:
- Integrates HTMX GET/POST/PUT/DELETE workflows
- Validates parameters (e.g. name length, unique codes)
- Updates UI using OOB HTML swaps

### CrudApiHandler (`CrudApiHandler.bas`)
Provides REST API capabilities:
- Matches REST endpoints (`GET /api/items`, `POST /api/items`, `GET /api/items/{id}`, `PUT /api/items/{id}`, `DELETE /api/items/{id}`)
- Validates XML/JSON payloads and outputs standardized HTTP response structures using `WebApiUtils`
